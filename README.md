# T88Extractor
Extract T88 File for PC-8801 Tape Data

PC-8001用の複数データがセーブされているT88ファイルを解析し、複数のファイルを抽出して個別のCMTファイルとして出力します。

対象はCSAVEされたデータとモニタのWコマンドで書かれたデータです。

T88ファイルになった時点でボーレートに関係なくデータ化されているので、1200bps改造などがされたPC-8001の保存ファイルでも扱えます。

扱えるファイル形式はCSAVEされたものとモニタのWコマンドで書かれたものです。Wコマンドで書かれたものにはファイル名がないので、"mon-開始アドレス"の名前が自動的に補われます。

テープは重複ファイル名が可能なので、ファイル名が重複した場追いは"(1)", "(2)"〜をファイル名に付加します。

CSAVE形式としてもモニタ形式としても認識できなかったデータは"$JUNK DATA"というファイルに格納されます。必要なら、ここかからデータを回収できます。ただし適切なデータを復元するのは全て自己責任です。

モニタ形式でチェックサムエラーなどが出た場合は、ファイル名にエラーメッセージが付加されます。その場合、エラー発生以後のデータは含まれません。

コマンドライン引数には任意の個数のT88ファイルのファイル名を記述できます。ワイルドカード可能です。

出力先は、T88ファイルのファイル名主部と同じ名前のディレクトリの中です。このディレクトリはT88ファイルを同じディウレク取りに作成されます。

オプション-listは、ファイル名のリストだけ行い、ファイルを作成しません。

オプション-overrideは出力ディレクトリの中味を事前に消去します。

本ツールはVisual Studio 2022 Version 17.4.0 Preview 2.0で.NET 7対応として開発されていますが、特別な機能は何も使用していないので、Visual Studio 2022と.NET 6でもビルドできると思います。バイナリーは特に提供しませんが、ソースはご自由にお使い下さい。

# N80MonCmt2Bin

　オマケです。N-BASICのモニタのWコマンドで書き出したファイル(CMT形式)から、ヘッダーやチェックサムの情報を除いたバイナリー値だけを抜き出します。ホスト側でマシン語を逆アセンブルする場合などに使います。チェックサムは確認していないので、正常に読み取れたファイルのみを使用する必要があります。

usage: N80MonCmtToBun SRC_FILE_NAME DST_FILE_NAME

御注意: 開始アドレス情報は取り除かれるので、手動でメモっておいて下さい。標準出力に文字列で表示します。出力バイナリーのオフセット0は、セーブの開始アドレスになります。
